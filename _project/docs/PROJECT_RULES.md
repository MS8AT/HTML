# Правила работы с сайтом

## Перед публикацией

1. Проверить статус:

```powershell
git status --short --branch
```

2. Проверить локальные ссылки, картинки, CSS `url(...)` и JSON-ссылки:

```powershell
python _project/tools/check_links.py
```

3. Если проверка нашла ошибку, сначала исправить её и запустить проверку повторно.

4. Открыть сайт локально и проверить основные страницы:

```powershell
python -m http.server 8000
```

Проверить минимум:

- `http://localhost:8000/index.html`
- `http://localhost:8000/ru/index.html`
- `http://localhost:8000/en/index.html`
- ключевые страницы из меню RU и EN

## Правило GitHub

Публиковать в GitHub можно только после проверки сайта.

Порядок:

1. Проверить ссылки и картинки.
2. Проверить главные страницы в браузере.
3. Посмотреть `git diff` / `git status`.
4. Сделать коммит с понятным сообщением.
5. Выполнить `git push`.
6. В ответе написать:

- что изменено;
- какие файлы перенесены или удалены из публичной части;
- какие проверки выполнены;
- хэш коммита;
- результат отправки на GitHub.

## Что не хранить в публичной части сайта

Не держать в корне и публичных папках `ru`, `en`, `images` файлы, которые не являются частью сайта:

- старые промпты и заметки;
- сканы и временные отчёты;
- генераторы и служебные скрипты;
- Excel-источники каталога;
- неиспользуемые черновые изображения;
- системные файлы `Thumbs.db`, `.DS_Store`, `Desktop.ini`;
- локальные настройки `.claude`, `.vscode`;
- пароли, токены и любые секреты.

Служебные материалы складывать в `_project`.
